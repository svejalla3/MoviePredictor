Introduction & Background

For our final project, our team will explore various methods with which we could determine genre from movie information. Our datasets are extracted from iMDb, a popular movie summary and rating website, that include movie poster images, movie titles, and accompanying metadata such as directors, actors, movie length, etc. Although movie datasets have been explored in Kaggle competitions and used as a benchmark to test various CNN and GAN architectures, we hope to [1] investigate an array of supervised and unsupervised methods to isolate the most effective techniques, and [2] experiment with engineering features that would rely on more than just the movie poster image.

Problem Definition

The ability to predict a movie's genre has a variety of applications, including the ability to propose films based on past viewing habits, viewer mood, etc., and has been extensively worked on. But, this could be developed even further.

As rightly said, “Life is about using your whole color palette” colors play an essential role in reflecting one’s opinion. So, can movie posters address the similarity of movie genres and recommend them to the user?

To reflect the content of a film, movie posters within a genre use a specific color scheme. For example, brighter and more playful (i.e., yellow, orange) colors depict comedy movies. Moreover, movie genres can be clustered into 19  main categories or combinations. Working with a final dataset of 27,602 movies spanning across various genres along with their posters, synopsis, budget, popularity, production company, actors, and other features, we aim to use this dataset to predict the genre of a movie.

In order to detect genre, we test a variety of supervised and unsupervised algorithms in this study. We concentrate on using the latent information in the movie poster for our supervised component. We use movie metadata for our unsupervised component to fit a variety of clustering algorithms. In our report, we give a thorough analysis and comparison of a variety of various approaches.

Data cleaning & Pre-processing

The TMDB API provides a comprehensive collection of data for a large number of movies, which consists of excess features like the number of global releases and translated languages that can be dropped from our dataframe for our initial analysis. This is not only because this information would be difficult to interpret from a machine learning perspective, but there are other more informative features present. As a result, we drop these features from the data that we obtain from the TMDB API.

Half of our final selection of features were adequately represented as 1D, stand-alone values. These include Budget, Popularity, Vote Count, Vote Average, Revenue, etc. For this type of information, a continuous linear representation is valid. For the indices of data such as keywords, however, `Fishing` which has an index of 3 is not greater than `Giant Insect`, which has an index of 1. This information had to be translated to a one-hot vector representation. The set of feature that required this kind of processing are:

  • Production companies
  • Keywords
  • Genres
  • Actors
  • Original language
  • Release date

For each feature in focus, the string values were standardized as integers using a dictionary of the original values. Some features (such as Actors and Keywords) had a variable number of values, therefore, we created two new features from such columns: one representing the top value and another representing all the values. While experimenting, we use one of these representations over the other when training various unsupervised model pipelines.

Fig 3: One-Hot Encoding Example

For features that had a very large set of unique values, i.e. would have a very large dictionary size to be used for one-hot processing, we would only assign a unique index to values that occurred above a certain threshold. Values that occurred less frequently were assigned to the “other” category and given the same index. This effectively decreased the dictionary size per feature, hence, decreasing the size of the final one-hot encoding of a movie.

Below are the features that were subject to this process, and the threshold values that were applied:

  • Actors: 40 occurrences
  • Production Companies: 100 occurrences
  • Key Words: 300 occurrences

We limited one-hot encoding to the variables most greatly affecting the movie genre. This equates to only producing one binary variable for top_genre_index, top_genere_indices, top_prod_company, all_prod_companies, all-keywords, actors_top3, actors_top2, actor_top1 and grouping all other labels into a new category. Therefore, by returning 1 or 0, respectively, the new dummy variables indicate whether or not one of the most prevalent labels is present for a given observation.

 

Fig 4: Converting the  original columns to one-hot encoded values

We obtain a dataset of size n x d using one-hot encoding, where n denotes the number of rows in the dataset and d, the number of distinct items in the category. A dataframe with dimensions of 27602 x 592 is obtained after concatenating the entire one-hot encoded dataset.

In order to generate a smaller but more effective dataset for our study, correlation is a crucial tool since it aims to ascertain the degree of association between variables. Dimensionality Reduction

The preprocessed data – after dropping unnecessary columns and encoding every remaining column to the appropriate format, was still huge with 554 columns and sparse due to the one-hot encodings. Not only is such high dimensional data very complicated to work with, it also could lead to overfitting. So, this presented us an opportunity to use dimensionality reduction and compress the data.

Dimensionality reduction is a class of transformation techniques that compress high dimensional data to lower dimensional representations while retaining as much information as possible. Having learnt the Principal Component Analysis (PCA) method recently in class, we used it to the desired effect. PCA transforms the data from its original basis vectors to a set of new orthogonal basis vectors which maximize the variance along their respective directions. Mathematically, these vectors are the eigenvectors of the covariance matrix and the variance is given by the eigenvalues. By choosing a subset of eigenvectors accounting for the most variance, we can reduce the dimensions.

For training our first set of models, we used PCA to get a 64 dimensional representation of our data. This explained a variance of 88.91% from the whole dataset.

Experimentation

For our final project, we hoped to explore numerous models to determine which would perform the best within the context of our problem. These models are listed below:

  1. GMM (generalized for hard-classification)
  2. K-means
  3. DBSCAN

For our first two experiments with unsupervised learning, we applied GMM and Kmeans to the IMDb movie dataset we processed. In order to be able to validate the performance of GMM, we built a function that would map component indices to the ground-truth genre indices we had in our data table. As discussed in class, the component/cluster index assignment by GMM/Kmeans is not constrained to match our labels at output. We map indices by:

  1. Collecting GMM predictions for a given genre class G
  2. Extracting the component index I that was assigned most frequently to this data sample
  3. Mapping G → I as long as I hasn’t already been mapped to a different genre and save this information in a dictionary
  4. Repeating for all genres in ground truth

The first experiments used only the features that were one-hot encoded to represent each movie. Each movie representation was then passed through PCA to reduce dimensionality to 64. Both GMM and Kmeans did not perform as well as we hoped in this initial run. We have found that this is a result of the following:

  1. Drastic class imbalance.
  • There are ~7,000 movies that have `Drama` as the primary genre, but only 166 movies have `History`.

  1. Movies can often be described by more than one genre.
  • Because a movie can have multiple genres at once, perhaps a hard clustering approach is not the best option for this problem.
  • This is most likely the reason you see less than 19 unique clusters in the Kmeans visualization (Fig 7) - many genres may be very similar to others.

DBSCAN Implementation & Evaluation

DBSCAN (Density-based spatial clustering of applications with noise) is a density-based clustering method that does not require explicit declaration of the number of clusters like KMeans or GMMs, but rather implicitly decides the optimal number of clusters based on the distribution of data and the hyperparameter choices. We noted this method to be particularly sensitive to the hyperparameter choices and hence was difficult to tune.

We discovered that the category drama was being predicted much more often. Drama was almost 60% of the entire data, while the other categories were rare. Clustering Metrics

The metrics that were computed for all three of our unsupervised models were:

  1. Rand Index: measures the similarity of two discrete label sets, ignoring permutations of the labels. This was a good choice because the cluster/component indices assigned by GMM, Kmeans, or DBSCAN need not match the label indices in ground truth.

  1. Silhouette Coefficient: calculated by comparing the mean distance between a sample and all other points in the same cluster, to the mean distance between a sample and all other points in the nearest cluster.

  1. Davies-Bouldin Index: defined as the average similarity between each cluster, that is computed by comparing the distance between cluster centroids to the distance between the cluster centroid and the points belonging to that cluster.

The team also experimented with calculating pairwise metrics (accuracy, weighted precision, weighted recall, and weighted f1-score), but the technique we developed to map ground truth labels to the labels assigned by models was unreliable due to class imbalance.

  • Mapping ground truth to predicted clusters/components: after an unsupervised model was fit, the predicted labels to trained data associated with ground truth label y were isolated. The most frequently appearing cluster/component label occurring in that set was mapped to ground truth label y. This was repeated across all labels in descending order of class share-of-voice, and the mapping was recorded given that the ground truth label has not been assigned already to a different cluster/component. Due to the class imbalance in our data, however, some clusters with the lowest share-of-voice were mapped to ground truth labels pretty randomly.

Clustering Visualization using t-SNE

t-distributed Stochastic Neighbour Embedding (t-SNE) is a non-linear dimensionality reduction technique mainly used for 2D or 3D visualizations of high dimensional data. T-SNE aims to embed similar points close to each other and dissimilar points further from each other. Unlike PCA, (i) this method is non-linear and (ii) it is iterative and cannot be transferred to unseen data. We use t-SNE to visualize the results of our clustering algorithms.

Conclusion

After evaluating the results across various combinations of features, computed for 3 different unsupervised models and 3 different metric algorithms, we found that different models performed best under different feature combinations. Furthermore, the same model could have yielded a higher Rand Index score with one feature combination, but a more optimal DB Index or Silhouette Coefficient score for another!

Generally, for Kmeans, GMM and DBSCAN, optimal Silhouette Coefficient and DB Index were derived from one feature space, and optimal Rand Index was derived from another. Silhouette Coefficient and DB Index both evaluate the quality of a clustering by computing and comparing the dispersion vs. cohesion of individual clusters. Within our problem space, this may not correlate with effective movie genre classification. The real-world distributions of movie genres cannot be neatly divided into distinct groups, and many movies may have attributes of multiple genres at one! How many times have we watched a Drama that was also a Thriller and a Romance? The Rand Index, on the other hand, can be interpreted as an accuracy metric in a setting where the ground truth labels are not aligned with cluster/component assignments. We can interpret the Rand Index being better on different feature selections that Silhouette and DB Index to mean that “better clusters” did not necessarily translate to better genre classification of movies.

The final Rand Index scores were relatively high for the models explored in this project (K-Means: 0.782, GMM: 0.757, DBSCAN: 0.579), indicating that our unsupervised clustering algorithms were able to resolve genre classification of movies decently well. However, our final metrics on the CNN-based Supervised approach were better. Given our observation that movies may not be easily separable into distinct genre groups, we recommend a supervised multi-label approach to extracting genre from movie poster, over training unsupervised clustering models on movie metadata for the purpose of genre assignment.
